# Roles & Personas — Responsibility Matrix

- **R** = Responsible (does the work)
- **A** = Accountable (final authority/ownership)
- **C** = Consulted (provides input)
- **I** = Informed (kept in the loop)

## Using the Matrix

1. **Initiation Phase**: PM is accountable; involves PdM, BA, and stakeholders
2. **Planning Phase**: PdM/BA are accountable for requirements; Dev estimates effort
3. **Execution Phase**: PM and SM drive cadence; Dev and QA execute; others track progress
4. **Release Phase**: PM and PdM coordinate; Dev and QA execute the release
5. **Retrospective**: SM facilitates; team participates; PM tracks improvements

---

## Key Principles

- **Clear Accountability**: One person is accountable for each outcome
- **Appropriate Delegation**: Responsible parties have authority and resources
- **Collaboration**: Consulted roles add necessary input before decisions
- **Transparency**: Informed roles stay updated on progress and risks

---

**Customize for your team**: This matrix represents the OctoAcme baseline. Adapt roles based on your project size, team structure, and risk profile.
