# Roles & Personas — Onboarding Checklist

Use this checklist to ensure all key project roles are assigned, responsibilities are understood, and handoffs are clear for each OctoAcme project.

## Pre-Project Setup

- [ ] **Project Manager assigned**
  - [ ] Understands coordination and timeline responsibilities
  - [ ] Familiar with project charter template

- [ ] **Product Manager / Product Lead assigned**
  - [ ] Understands backlog prioritization responsibilities
  - [ ] Aligned on success metrics

- [ ] **Developer(s) assigned**
  - [ ] Know technical approach and Definition of Done
  - [ ] Understand PR and code review standards

- [ ] **QA/Testing assigned**
  - [ ] Understands acceptance criteria and test plan approach
  - [ ] Familiar with Definition of Done checklist

- [ ] **UX/UI Designer assigned (if applicable)**
  - [ ] Understands design review and handoff process
  - [ ] Knows collaboration points with Dev and Product

- [ ] **Business Analyst assigned (if applicable)**
  - [ ] Understands requirement elicitation and documentation approach
  - [ ] Knows clarification and escalation paths

- [ ] **Scrum Master / Process Facilitator assigned (if applicable)**
  - [ ] Understands ceremony facilitation responsibilities
  - [ ] Knows team health and metrics tracking approach

- [ ] **Stakeholders identified**
  - [ ] Communication plan established
  - [ ] Decision rights clarified
  - [ ] Update frequency agreed (weekly, milestone-based, etc.)

## Team Alignment & Kickoff

- [ ] **Project Charter/One-pager completed**
  - [ ] Problem statement clear
  - [ ] Success metrics defined
  - [ ] Timeline and milestones agreed

- [ ] **Team kickoff meeting held**
  - [ ] All roles introduced and responsibilities reviewed
  - [ ] Communication cadence agreed (standups, syncs, demos)
  - [ ] Escalation paths communicated

- [ ] **Documentation & Access**
  - [ ] All team members have access to `docs/` folder
  - [ ] Project board / tracking system set up
  - [ ] Risk register initialized
  - [ ] Relevant Copilot Space contexts configured (if applicable)

- [ ] **Processes & Standards Reviewed**
  - [ ] Definition of Done documented and agreed
  - [ ] PR and code review standards understood
  - [ ] QA and testing approach clarified
  - [ ] Release process understood

- [ ] **Handoff Points & Dependencies Clarified**
  - [ ] Design → Development handoff documented
  - [ ] Requirements → Development clarified
  - [ ] Development → QA process established
  - [ ] Cross-team dependencies identified

## Ongoing (Throughout Project)

- [ ] **Weekly PM/PdM sync scheduled**
  - [ ] Risk and dependency review
  - [ ] Progress tracking
  - [ ] Stakeholder communication planning

- [ ] **Team ceremonies on regular cadence**
  - [ ] Daily standups (15 min)
  - [ ] Sprint planning (or equivalent)
  - [ ] Demo/Review sessions
  - [ ] Retrospectives (sprint or milestone-based)

- [ ] **Regular role check-ins**
  - [ ] Ensure role clarity is maintained
  - [ ] Address any gaps or overlaps
  - [ ] Adjust as needed based on project evolution

- [ ] **Documentation kept current**
  - [ ] Risk register updated weekly
  - [ ] Project artifacts (charter, roadmap, backlog) maintained
  - [ ] Lessons learned captured for retro

---

## Notes

- Adjust role assignment based on project size, complexity, and team structure
- Smaller projects may combine roles (e.g., PM + Scrum Master)
- Larger initiatives may have multiple developers, QA engineers, or designers
- Always clarify decision rights and escalation paths upfront to avoid confusion
